1. Методы dtbase:
Select - делает выборку объектов из файла и отдает их по очереди в fn.
  fn возвращает объект, который попадает под условие.
  В результате возвращается массив, в котором может не быть элементов. Массив
  возвращается пользователю.

Update - как и Select, данный метод делает выборку объектов из базы данных.
  В отличие от Select, при передаче объектов в fn, все они должны быть
  возвращены. fn вносит изменения в нужный объект и возвращает его.
  Если fn не возвращает объект, то это вызывает ошибку.
  В результате возвращается массив со всеми элементами, которые и были. Данный
  массив не возвращается пользователю, а записывается в файл. Файл
  перезаписывается.
  Ответ из базы при успехе 'ок'

Insert - как и Select, данный метод делает выбоку объектов. В fn биндится
  условие. При итерации, проводится проверка элемента на это условие и если
  совпадение, то элемент не возвращается, что вызывает ошибку. Например если
  такой id уже есть, то записить не будет добавлена.
  Если все элементы были возвращены, значит условие не сработало и объект можно
  добавлять.
  Объект передается в options в поле insert. Он же биндится на fn и будет
  сверятся при итерации.
  После окончания итерации всех элементов объект добавлятеся последним.
  В результате возвращается новый массив элементов. Он как и update перезаписывается
  в файл.
  Ответ из базы при успехе 'ок'

Delete - как и Select, данный метод делает выборку объектов из указанной таблицы.
  В fn биндится переданный объект с условием. Во время итерации fn запускается с
  очередным item и внутри происходит сверка с переданным объектом. Если условие
  не подходит под объект, то объект возвращается. В итоге будет создан новый
  массив из item, которые не попали под условие. Этот массив будет записан в
  базу.
  То что попадет под условие, не будет возвращено функцией fn как результат и
  будет потеряно. 
--------------------------------------------------------------------------------
2.  Options - объект.
    options.select - объект, по которому будет произведена выборка
    options.insert - объект, который будет добавлен
    options.update - объект, который содержит информация для сравнения и
      обновления.
    Если данные свойства не переданы при вызове ментодов, то будет вызвана
      ошибка.
-----------------------------------------------------------------------------------
3. Unique - если в таблице у вас есть уникальное поле, то вам нужно при использовании
  метода Insert всегда передавать условие по проверке на это поле. Иначе
  вы получите дубликаты в базе.
